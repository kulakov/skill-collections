# PRD Review Checklist

Проведи ревью PRD документа.

Если указан файл ($ARGUMENTS), прочитай его. Если нет — спроси какой PRD ревьюить.

Проверь PRD по этому чеклисту и дай оценку каждому пункту (✅ / ⚠️ / ❌):

## Чеклист

### Problem & Context
- [ ] Проблема чётко сформулирована
- [ ] Есть данные подтверждающие проблему
- [ ] Понятно почему решаем сейчас

### Solution
- [ ] Решение понятно описано
- [ ] Есть user stories или use cases
- [ ] Scope чётко определён (что включено / что нет)

### Success Metrics
- [ ] Определены измеримые метрики успеха
- [ ] Есть baseline и target
- [ ] Метрики aligned с бизнес-целями

### Technical & Feasibility
- [ ] Учтены технические constraints
- [ ] Есть оценка сложности
- [ ] Dependencies идентифицированы

### Risks
- [ ] Риски идентифицированы
- [ ] Есть план митигации

## Output Format

**Общая оценка:** [🟢 Ready / 🟡 Needs Work / 🔴 Major Issues]

**Сильные стороны:**
- [что хорошо]

**Требует доработки:**
- [что улучшить + конкретные предложения]

**Критичные проблемы (если есть):**
- [что блокирует]

**Рекомендация:** [Ship / Iterate / Rethink]
