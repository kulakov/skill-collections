# Landing Page Methodology

Методика создания конверсионного лендинга: структура, копирайтинг, фреймворки.

## Входные данные

**Продукт/сервис:** $ARGUMENTS

---

## ПРИНЦИП: Один лендинг = Одна цель

- Одна страница, одно действие, один CTA
- Каждый блок ведёт к одному и тому же действию
- Всё, что не ведёт к действию — убрать

---

## ФРЕЙМВОРКИ КОПИРАЙТИНГА

### StoryBrand (Donald Miller) — основной фреймворк

```
1. ГЕРОЙ (клиент) — кто он и чего хочет
2. ПРОБЛЕМА — внешняя, внутренняя, философская
3. ПРОВОДНИК (ты) — эмпатия + авторитет
4. ПЛАН — 3 простых шага
5. ПРИЗЫВ — прямой CTA + переходный CTA
6. ПРОВАЛ — что будет если НЕ действовать
7. УСПЕХ — как изменится жизнь
```

**Ключ:** Клиент — герой, ты — проводник. Не ты герой.

### PAS — для секций с болью

```
P (Problem)  — назови боль точно как клиент её описывает
A (Agitate)  — покрути нож: покажи последствия, потери, упущенное
S (Solution) — представь решение как облегчение
```

### AIDA — для общей структуры потока

```
A (Attention) — хук, заголовок (5 секунд на захват)
I (Interest)  — релевантная информация, "это про меня"
D (Desire)    — выгоды + доказательства + визуализация результата
A (Action)    — CTA с минимальным трением
```

### Before → After → Bridge — для коротких блоков

```
Before: Как сейчас (боль, проблема)
After:  Как будет (мечта, результат)
Bridge: Продукт = мост между ними
```

## СТРУКТУРА ЛЕНДИНГА (10 секций)

### 1. Hero Section (above the fold)

**Задача:** За 5 секунд ответить на 3 вопроса:
1. Что это?
2. Для кого?
3. Почему мне это нужно?

**Элементы:**
- **Headline** — ВИСП-заголовок (Причастность + Выгода)
- **Subheadline** — уточнение: кому, как, за сколько
- **Primary CTA** — одна кнопка, одно действие
- **Hero image/video** — продукт в контексте использования
- **Social proof lite** — "1200+ клиентов" / "рейтинг 4.9"

**Формула заголовка:**
```
[Результат, который хочет клиент] без [главное возражение/страх]
```

**Антипаттерны:**
- "Мы лучший сервис для..." (герой — ты, а не клиент)
- "Инновационная платформа..." (фичи вместо выгод)
- "Добро пожаловать!" (ноль информации)

### 2. Logo Bar / Trust Strip

**Задача:** Мгновенное социальное доказательство

- 4-8 логотипов клиентов / партнёров / СМИ
- "Нам доверяют:" / "Как видели в:" / "Клиенты:"
- Серые логотипы, без ссылок (не уводить со страницы)

### 3. Problem Section (PAS — Problem + Agitate)

**Задача:** Клиент должен сказать "Да, это про меня!"

**Структура:**
1. Назови проблему словами клиента (не своими)
2. Покажи последствия бездействия (agitate)
3. Покажи что текущие решения не работают

**Формат:** 2-3 абзаца или 3 карточки проблем

**Приём:** "Знакомо?" / "Если вы..." — прямое обращение

### 4. Solution Section

**Задача:** Показать продукт как ответ на боль из секции 3

**Структура:**
1. One-liner: что делает продукт (Before -> After -> Bridge)
2. Скриншот / демо / визуал продукта
3. 1-2 предложения: чем отличается от альтернатив

**Не перегружать:** это мост, не каталог фич

### 5. Benefits (не Features!)

**Задача:** Показать ВЫГОДЫ, не характеристики

**Формат:** 3-5 карточек, каждая:
- Иконка (SVG, не эмодзи)
- Заголовок-выгода: "Экономьте 10 часов в неделю"
- 1-2 предложения: как конкретно

**Трансформация фичи в выгоду:**
```
Фича:  "AI-аналитика в реальном времени"
Выгода: "Узнавайте о проблемах до того, как их заметят клиенты"
```

### 6. How It Works (План из StoryBrand)

**Задача:** Снять тревогу "а как это работает?"

**Формат:** 3 шага (не больше!)
```
Шаг 1: [Простое действие]      → "Оставьте заявку"
Шаг 2: [Что происходит]        → "Мы настроим за 24 часа"
Шаг 3: [Результат для клиента] → "Получайте отчёты каждое утро"
```

**Принцип:** Максимальная простота. "Это легко" > "Это мощно"

### 7. Social Proof (развёрнутое)

**Задача:** Доказать что это работает

**Типы доказательств (от сильного к слабому):**
1. Кейсы с цифрами: "Компания X увеличила конверсию на 47%"
2. Видео-отзывы (15-30 сек)
3. Текстовые отзывы с фото + должностью
4. Метрики: "10,000+ пользователей", "обработано 1M заявок"
5. Логотипы клиентов
6. Рейтинги (G2, Capterra, отзовики)

**Формат отзыва:**
```
"[Конкретный результат в цифрах]"
— Имя, Должность, Компания
```

### 8. Pricing / Offer

**Задача:** Чётко показать что получает клиент и за сколько

**Принципы:**
- Якорь: показать ценность ДО цены ("Экономия 120 часов/год")
- Сравнение: 2-3 тарифа, средний = рекомендуемый
- Risk reversal: гарантия / бесплатный триал / money-back

**Если нет публичных цен:**
- "Получить предложение" с калькулятором или формой
- Показать ROI вместо цены

### 9. FAQ (Обработка возражений)

**Задача:** Закрыть оставшиеся сомнения

**5-8 вопросов из категорий:**
1. Безопасность / данные
2. Интеграции / совместимость
3. Сроки / онбординг
4. Отличие от конкурентов
5. Гарантии / отмена

**Принцип:** Каждый FAQ — скрытое возражение. Формулируй вопрос как его задал бы скептик.

### 10. Final CTA (закрывающий блок)

**Задача:** Последний толчок к действию

**Структура:**
1. Recap выгоды (1 предложение)
2. Urgency (если есть и не фейковый): дедлайн, лимит, бонус
3. CTA-кнопка (та же, что в Hero)
4. Micro-copy под кнопкой: "Бесплатно" / "Без карты" / "2 минуты"

## КОНВЕРСИОННЫЕ ПРИНЦИПЫ

### Визуальная иерархия
- F-паттерн чтения: важное слева-сверху
- Один CTA-цвет на всю страницу (контрастный)
- Белое пространство > перегруженность

### Копирайтинг
- Конкретика > абстракция: "47%" > "значительно"
- "Вы/Ваш" > "Мы/Наш": соотношение минимум 2:1
- Один абзац = одна мысль, max 3 строки
- Активный залог: "Получите отчёт" > "Отчёт будет сгенерирован"

### Доверие
- Реальные фото > стоковые
- Имя + должность + компания > анонимный отзыв
- Конкретные цифры > общие утверждения
- Логотипы > слова "нам доверяют крупные компании"

### Скорость
- CTA выше сгиба (первый экран)
- Минимум полей в форме (имя + email достаточно)
- Sticky CTA на мобильных
- Прогресс-бар если форма длинная

### Мобайл
- 60-70% трафика с мобильных
- Текст читаемый без зума (16px+)
- Кнопки 44x44px+, большие tap targets
- Один столбец, вертикальный скролл

## ЧЕКЛИСТ ПЕРЕД ФИНАЛИЗАЦИЕЙ

- [ ] Заголовок отвечает на "что, для кого, зачем" за 5 секунд
- [ ] CTA одинаковый и повторяется 3-4 раза на странице
- [ ] Нет внешних ссылок (кроме соцсетей в футере)
- [ ] Social proof конкретный (цифры, имена, компании)
- [ ] FAQ закрывает топ-5 возражений
- [ ] Мобильная версия проверена
- [ ] Скорость загрузки < 3 секунд
- [ ] Форма минимальна (2-3 поля)
- [ ] "Вы" > "Мы" по всему тексту

## ФОРМАТ ВЫВОДА

Для каждой секции выдай:
1. **Заголовок секции**
2. **Текст** (готовый к использованию)
3. **Визуальные заметки** (что показать, какой layout)
4. **CTA** (текст кнопки + micro-copy)

В конце — полный wireframe-текст лендинга от Hero до Final CTA.
