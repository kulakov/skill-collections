# Track Prep — подготовка к сессии

$ARGUMENTS — имя команды. Если пусто — спроси.

## Процесс

### Шаг 1: Оркестратор (цепочка блокеров + контракт)

Запусти `/track-orch $ARGUMENTS` — получи:
- Цепочку блокеров (от текущего состояния до эндгейма)
- Контракт сессии (всегда = "снимаем блокер 0")
- Оценку гипотез (есть ли Type B/C?)

### Шаг 2: Трекер (вопросы)

Запусти субагента-Трекера (Task tool, промпт: `~/.claude/skills/track/agents/tracker.md`) с задачей PREP:

Входные данные:
- STATE.md команды (включая цепочку блокеров)
- Последняя запись DIARY.md
- Контракт сессии от Оркестратора (блокер 0)

Задание:
- Подготовить вопросы по блокеру 0 (снято? что мешает?)
- Подобрать фреймворк из `~/.claude/skills/track/frameworks/`
- Проверить actions с прошлой сессии

### Шаг 3: Супервизор (мета-анализ)

Запусти субагента-Супервизора (Task tool, промпт: `~/.claude/skills/track/agents/supervisor.md`) с задачей PREP:

Входные данные:
- STATE.md (полный)
- Контракт сессии

Задание:
- Мета-картина (N/12 сессий, % прогресса)
- Алерты (застревание, drift, цикличность)
- Проверка связи контракта с целью трека

### Шаг 4: Собрать PREP BRIEF

```markdown
# Prep Brief: [Команда] — Сессия N
Дата: YYYY-MM-DD

## Цепочка блокеров (актуальная)
[От Оркестратора — полная цепочка от блокера 0 до эндгейма]

## Контракт сессии
Режим: [Bottleneck Engine / Contradiction Resolver]
[Формулировка — всегда про блокер 0]

## Оценка гипотез
[Есть ли Type B/C? Если только A — предложить 4 инверсии]

## Ожидаемые изменения с прошлого раза
- [ ] [Что команда должна была сделать]

## Вопросы для сессии
[От Трекера — вопросы привязаны к блокеру 0]

## Рекомендуемый фреймворк
[От Трекера]

## Алерты Супервизора
[Если есть]

## Референсы
[Если были найдены]
```
