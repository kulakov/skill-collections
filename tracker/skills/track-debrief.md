# Track Debrief — разбор после сессии

$ARGUMENTS — имя команды. Если пусто — спроси.

## Входные данные

Lance загружает транскрипт Granola и/или дневник Paper. Если не загрузил — спроси.

## Процесс

### Шаг 1: Норма (файлы)

- Найти новый транскрипт (проверить `gh api repos/kulakov/granola-notes/contents/notes`)
- Сохранить в `teams/<team-name>/transcripts/session-NN_YYYY-MM-DD.md`
- Если есть Paper дневник — в `notes/`

### Шаг 2: Трекер (анализ транскрипта)

Запусти субагента-Трекера (Task tool, промпт: `~/.claude/skills/track/agents/tracker.md`) с задачей DEBRIEF:

Входные данные:
- Транскрипт/дневник
- STATE.md команды
- Контракт сессии из BOTTLENECK_LOG.md (активное ограничение)

Задание:
- Извлечь решения, гипотезы, инсайты, action items
- Определить: изменилось ли узкое место?
- Предложить Светофор (1-5) и Спидометр
- Оценить: снято ли ограничение из контракта?

### Шаг 3: Супервизор (мета-анализ)

Запусти субагента-Супервизора с задачей DEBRIEF:

Входные данные:
- STATE.md + анализ от Трекера

Задание:
- Тренд-анализ (как сессия вписывается в общую картину)
- Алерты (паттерны, drift)
- Прогноз (успеем ли закрыть контракт трека)

### Шаг 4: Оркестратор (проверка контракта + обновление цепочки)

Прочитать `~/.claude/skills/track/agents/orchestrator.md` и выполнить DEBRIEF:

- Контракт выполнен? (ДА / НЕТ / ЧАСТИЧНО)
- **Обновить цепочку блокеров:**
  - Блокер 0 снят? → сдвинуть цепочку (блокер 1 становится блокером 0)
  - Не снят? → зафиксировать причину, скорректировать подход
  - Появился новый блокер? → вставить в цепочку
- Обновить BOTTLENECK_LOG.md
- Обновить CONTRADICTIONS.md (если был режим ТРИЗ)
- **Оценить гипотезы:** есть ли Type B/C? Если только A — предложить 4 инверсии
- Сформулировать предварительный контракт следующей сессии (= новый блокер 0)

### Шаг 5: Обновить STATE.md

- **Цепочка блокеров** (обновлённая)
- Узкое место (детализация нового блокера 0), North Star Metric, гипотезы
- Светофор + Спидометр (добавить строки)
- Ключевые инсайты

### Шаг 6: Добавить запись в DIARY.md

По шаблону `~/.claude/skills/track/templates/DIARY-ENTRY.md`

### Шаг 7: Показать Lance итог

```markdown
## Результат сессии N

### Контракт: [ДА/НЕТ/ЧАСТИЧНО]
[Что произошло]

### Цепочка блокеров (обновлённая)
**Блокер 0 (СЕЙЧАС):** [тот же / новый]
Снимаем: [...]
Критерий: [...]

↓

**Блокер 1:** [...]
...

↓

**Эндгейм:** [...]

### Оценка гипотез
[Есть ли Type B/C? Если только A — предложить 4 инверсии]

### Светофор: [N] / Спидометр: [метрика] = [значение]

### Следующий контракт (предварительный)
Режим: [Bottleneck Engine / Contradiction Resolver]
[Формулировка — всегда про новый блокер 0]

### Алерты Супервизора
[Если есть]
```

Спросить Lance:
- Согласен с оценкой Светофора?
- Правильно ли определено узкое место?
- Что добавить/исправить?
