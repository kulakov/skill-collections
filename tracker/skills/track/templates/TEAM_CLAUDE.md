## Поведение по умолчанию

### При получении нового файла

Когда Lance кидает файл в папку или упоминает файл:

1. **Определи тип:**
   - Транскрипт Granola (есть `**Time:**`, `**Attendees:**`) -> полный debrief pipeline
   - Документ/таблица (.md, .xlsx, .csv, .docx) -> анализ через призму bottleneck
   - Картинка (.jpg, .png) -> описать через субагент, привязать к контексту
   - Другое -> спросить зачем

2. **Транскрипт -> Debrief pipeline:**
   - Скопировать в `11-TRACKER/teams/[TEAM_SLUG]/transcripts/session-NN_YYYY-MM-DD.md`
   - Извлечь: решения, гипотезы, инсайты, action items
   - Оценить Светофор (предложить Lance)
   - Обновить Спидометр
   - Определить: изменился ли bottleneck?
   - Записать в DIARY.md
   - Обновить STATE.md
   - Показать итог в формате Нормы

3. **Документ -> Анализ:**
   ```
   Как этот документ помогает снять текущее ограничение?

   Текущий bottleneck: [CURRENT_BOTTLENECK]

   -- Прямая связь: [что в документе относится к bottleneck]
   -- Косвенная связь: [что полезно, но не про bottleneck]
   -- Рекомендация: [что делать с этой информацией]
   -- Метод из каталога: [какой метод применить]
   ```

### При вопросе "что делать"

1. Прочитать STATE.md -> текущий bottleneck
2. Посмотреть DIARY.md -> последние actions
3. Предложить следующий шаг через призму ТОС
4. Рекомендовать метод из METHODS_CATALOG.md

### При просьбе подготовиться к сессии

Запустить `/track-prep [TEAM_SLUG]`

### При просьбе разобрать сессию

Запустить `/track-debrief [TEAM_SLUG]`

## Правила

1. **Всегда через bottleneck.** Любой анализ начинай с текущего ограничения из STATE.md
2. **Не придумывай данные.** Если не знаешь -- спроси Lance или скажи "не знаю"
3. **Методы из каталога.** При анализе проверяй METHODS_CATALOG.md -- какой метод подходит
4. **Норма молча.** Файловые операции делай без лишних вопросов, отчитывайся кратко
5. **Связь с целью.** Каждое действие должно быть связано с целью трека
6. **Изображения через субагент.** Картинки читай через Task tool с subagent_type="general-purpose"
