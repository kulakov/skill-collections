# JTBD (Jobs to be Done) -- Шаблон

## Формула

```
Когда [ситуация],
я хочу [мотивация],
чтобы [ожидаемый результат]
```

## Ключевые вопросы

### Понимание работы

1. **За какие изменения в своей жизни клиент готов платить?**
2. **Что клиент "нанимает" наш продукт сделать?**
3. **Насколько созрел спрос?** (Уже ищет решение vs. еще не осознал проблему)

### Текущее поведение

4. **Что клиент уже делает для решения этой задачи?**
5. **Какие продукты/сервисы уже "нанимает"?**
6. **Что не устраивает в текущем решении?**
7. **Сколько усилий тратит?**

### Признаки клиента

8. **Как отличить нашего клиента от не нашего?**
   - Квалифицирующие признаки (должны быть)
   - Отсекающие признаки (не должны быть)

### Сила переключения

9. **Что толкает от текущего решения?** (Push)
10. **Что тянет к новому?** (Pull)
11. **Что привязывает к старому?** (Habit)
12. **Что пугает в новом?** (Anxiety)

## Фреймворк "Зазор -> Барьер -> Ограничение"

### Зазор
Разрыв между желаемой и текущей действительностью.
- **Сила зазора:** насколько сильно хочет изменить жизнь?
- **Текущие усилия:** сколько уже тратит на это?

### Барьер
Почему зазор существует?
- Что мешает людям делать свою работу нормально?
- НЕ "нет системы" (это не барьер, это отсутствие решения)

### Ограничение
Почему барьер не могут убрать?
- Организационные, программные, экономические факторы
- Психологические барьеры (страх, привычка, статус)

### Решение
Как снимаем ограничение?
- Почему раньше нельзя было, а теперь можно?

### Ценность
Что получит бизнес?
- В деньгах, времени, качестве (конкретно!)

## Типичные ошибки

1. Описывают барьер через отсутствие решения ("нет CRM" -- это не барьер)
2. В ограничении рассказывают историю, а не диагноз
3. Путают барьер с ограничением (одно и то же разными словами)
4. Решение не объясняет "почему сейчас"
5. Ценность описана как "увеличение эффективности" без цифр

## Клиент != Пользователь

- **Клиент** -- держатель бюджета
- **Пользователь** -- взаимодействует с системой
- Выгодоприобретателями должны быть ОБА
