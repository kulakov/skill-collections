# Воронка гипотез -- методика и шаблон

## Принцип

Сравниваем гипотезы через единую воронку: каждая гипотеза меняет 1-2 конверсии, модель пересчитывает сквозную. **Самая сильная гипотеза = наименьший требуемый охват для достижения цели.**

Фреймворк выявляет:
- Разрыв между целевым и текущим состоянием
- Какой шаг воронки лимитирует сильнее всего
- Гипотезы типа B/C (убирают шаги) vs Type A (улучшают коэффициент)

---

## Процесс

> **Режим по умолчанию: Трекшн-карта 2.0 (Distribution First).**
> Классическая воронка — ниже, для deep tech / enterprise / regulated markets.
> Источник истины: `11-TRACKER/TRACTION_MAP_v2.md`

### Шаг 1: Построить воронку

Определить шаги от первого касания до регулярного использования.

**v2.0 (Distribution First) — по умолчанию:**

Над классической воронкой добавляется "pre-funnel" — дистрибуция:

```
[PRE-FUNNEL] Канал/база → Подписались → Вовлечены → Озвучили боль
[FUNNEL]     Увидели предложение → Попробовали MVP → Первый вау → Заплатили → Вернулись → Регулярно
```

Pre-funnel строится ДО продукта. Воронка начинается, когда есть канал с вовлеченной аудиторией.

**v1.0 (классика) — для deep tech / enterprise / regulated:**

```
Увидели рекламу → Лендинг → Заявка → Встреча → Установка → Первый вау → Вернулись → Регулярно
```

Между каждой парой шагов -- конверсия (C1, C2, ..., CN).

**Правила (обе версии):**
- Шаги должны быть наблюдаемыми (можно измерить)
- Конверсии между 0 и 1
- Количество шагов = специфика продукта (обычно 5-8)

### Шаг 2: Обратный счёт от цели (строка "Целевое")

1. Зафиксировать **цель**: N регулярных пользователей (или продаж, или revenue)
2. Поставить **лучшие обоснованные конверсии** на каждом шаге
3. Считать обратно: каждый предыдущий шаг = следующий / конверсия
4. Получить **требуемый охват** и **сквозную конверсию**

Это идеальный сценарий -- минимально необходимый охват при лучших конверсиях.

### Шаг 3: Текущее состояние (строка "Текущее")

1. Та же цель
2. **Реальные текущие конверсии** (из данных, не из фантазий)
3. Посчитать обратно

Разрыв "Целевое vs Текущее" = масштаб проблемы. Если требуемый охват нереалистичен -- текущая модель не работает.

### Шаг 4: Гипотезы (прямой счёт)

Каждая гипотеза -- отдельная строка воронки. Правила:
- **Меняется 1-2 конверсии** относительно "Текущего"
- Остальные конверсии остаются как в "Текущем"
- Модель пересчитывает: сколько охвата нужно? какая сквозная?

**Метрика сравнения:**

| Метрика | Что показывает | Формула |
|---------|---------------|---------|
| Сквозная | Общая эффективность воронки | Цель / Охват |
| Требуемый охват | Масштаб усилий | Обратный расчёт от цели |
| Конверсия от заявки | Эффективность после привлечения | Произведение конверсий от заявки |

**Чем меньше требуемый охват (= выше сквозная) -- тем сильнее гипотеза.**

### Шаг 5: Альтернативные воронки (Type B/C)

Самый мощный ход -- **убрать шаги из воронки**. Создать отдельную секцию:

```
Увидели → Лендинг → Скачали → Установили → Вернулись → Регулярно
```

Даже при низких конверсиях на оставшихся шагах, убирание 2-3 шагов даёт 10-100x улучшение сквозной.

**"Было бы классно"** -- бенчмарк: самая короткая возможная воронка с идеальными конверсиями. Показывает потолок.

### Шаг 6: Приоритизация

Дополнить каждую гипотезу:
- **Стоимость** (в неделях разработки)
- **Что измеряем** (конкретная метрика)
- **Дней до результата** (когда узнаем)
- **Источник** (откуда гипотеза)

Приоритет = Сила гипотезы (сквозная) / Стоимость.

## Связь с типами гипотез

| Тип | Что делает в воронке | Эффект на сквозную |
|-----|---------------------|-------------------|
| A | Улучшает 1 конверсию | +20-40% |
| B | Убирает/склеивает шаги | 2-10x |
| C | Другая воронка целиком | 10x+ |

Если в таблице только Type A (улучшение одной конверсии) -- генерировать Type B/C через 4 инверсии:
1. "Что если 0 людей на этом шаге?"
2. "Что если продукт распространяется через работу пользователя?"
3. "Что если мы встраиваемся в чужой поток?"
4. "Что если пользователь не принимал решения начать?"

## Формулы (для Excel)

### Структура колонок

```
ABS | RATE | ABS | RATE | ABS | RATE | ... | TARGET | THROUGH
```

Абсолютные числа (ABS) чередуются с конверсиями (RATE).

### Входные данные (переменные)

- **Конверсии** (RATE): числа от 0 до 1, вводятся вручную
- **Цель** (TARGET): фиксированное число, вводится вручную

### Вычисляемые значения (формулы)

Обратный расчёт от цели:

```
ABS[N-1] = ABS[N] / RATE[N-1]
ABS[N-2] = ABS[N-1] / RATE[N-2]
...
ABS[0] = ABS[1] / RATE[0]

THROUGH = TARGET / ABS[0]
```

### Self-serve строки (пропущенные шаги)

Когда шаг не применим, ячейки помечаются "--". Формула перепрыгивает:

```
Если RATE[i] = "--", пропустить ABS[i]
Ближайший непустой ABS слева = ближайший непустой ABS справа / первый непустой RATE между ними
```

```markdown

|---|-----|-------------|-------------|

| 2 | [Лендинг/сайт] | [...] | [...] |
| ... | | | |
| N | [Целевое действие] | [...] | [...] |

### Базовые сценарии

| Сценарий | C1 | C2 | ... | CN | Охват | Сквозная |
|----------|----|----|-----|----|-------|----------|
| Целевое  | [лучшие] | | | | = формула | = формула |
| Текущее  | [реальные] | | | | = формула | = формула |

### Гипотезы (полная воронка)

| # | Гипотеза | Что меняет | Тип | C_изм | Охват | Сквозная | Стоимость | Что измеряем |
|---|----------|-----------|-----|-------|-------|----------|-----------|-------------|
| 1 | [...] | C[X] | A/B/C | [...] | = формула | = формула | [нед] | [...] |

### Гипотезы (альтернативные воронки)

| # | Гипотеза | Воронка | Шаги | Охват | Сквозная | Тип |

### Приоритет

| # | Гипотеза | Сквозная | Стоимость (нед) | Сила/Стоимость | Решение |

| 1 | [...] | [...] | [...] | = формула | ДЕЛАТЬ / ОТЛОЖИТЬ / УБРАТЬ |
```

## Когда применять

- После первых 10-20 клиентских интервью (есть данные для конверсий)
- Когда команда генерирует гипотезы и нужно сравнить их силу
- Когда текущая модель не даёт нужный масштаб (охват нереалистичен)
- На дебрифе: обновить конверсии из реальных данных, пересчитать

## Антипаттерны

- **Все конверсии выдуманы** -- хотя бы "Текущее" должно быть из данных
- **Только Type A гипотезы** -- если ни одна не меняет структуру воронки, сила ограничена
- **Гипотезы меняют 3+ конверсий** -- это не одна гипотеза, а комбо; разбить
- **Нет строки "Было бы классно"** -- без неё не видно потолок
