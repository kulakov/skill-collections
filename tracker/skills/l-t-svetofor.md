# Светофор + Спидометр

Ты -- инструмент быстрой оценки прогресса трекинговой сессии. Запускается за 10 минут до конца сессии.

## Процесс

### 1. Определи команду

Если аргумент не указан -- спроси какая команда. Команды:

| Slug | Папка |
|------|-------|
| первичка | `01-WORK/Контур/! Первичка/` |
| единая-среда | `01-WORK/Контур/! единая среда бухгалтера /` |
| метапродукт | `01-WORK/Контур/! Метапродукт в корпоративном сегменте /` |
| росреестр | `01-WORK/Контур/! Поиск новых аудиторий для существующей функциональности /` |
| толк | `01-WORK/Контур/! Толк через промо-период /` |

### 2. Загрузи спидометр

Прочитай `СПИДОМЕТР.md` из папки команды.

### 3. Покажи текущее состояние

```
== СПИДОМЕТР: [название команды] ==

Метрика: [что измеряем]
Было: [значение на прошлой сессии]
Сейчас: [текущее значение -- спроси у Lance если не знаешь]
Цель: [целевое значение]
Дедлайн: [когда]
```

### 4. Спроси светофор

Шкала:

| Балл | Значение |
|------|----------|
| 5 | Команда работала на цель и достигла всего чего хотела |
| 4 | Команда работала на цель и серьезно продвинулась |
| 3 | Команда работала на цель и слабо продвинулась |
| 2 | Команда работала не на цель |
| 1 | Ничего не было |

Спроси Lance:
1. Какой балл светофора?
2. Новое значение спидометра (если изменилось)?
3. Короткий комментарий (1 предложение)

### 5. Обнови файл

Добавь строку в таблицу "История светофора" в `СПИДОМЕТР.md`:

```
| [дата] | [номер сессии] | [балл] | [спидометр] | [комментарий] |
```

### 6. Покажи тренд

Если есть 3+ записей -- покажи тренд:
- Средний светофор за последние 3 сессии
- Динамика спидометра (растет/стоит/падает)
- Если средний светофор < 3 три сессии подряд -- алерт: "Команда буксует. Рассмотреть смену подхода или остановку трека."
