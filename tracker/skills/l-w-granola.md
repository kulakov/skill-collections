# Универсальный анализ встречи

Проанализируй транскрипт встречи и создай полный структурированный разбор.

## Порядок работы

### 1. ЛИД (3-4 предложения)

Напиши сжатую формулировку: что произошло на встрече, какой главный результат, что изменилось. Лид должен быть понятен без контекста — как заголовок новости. Начинай с самого важного.

### 2. Участники и их предполагаемые цели

Для каждого участника:
- Имя и роль (если понятна из контекста)
- Что он хотел получить от встречи (цель)
- Получил ли он это (да/частично/нет)

### 3. Все тезисы

Извлеки ВСЕ значимые высказывания. Каждый тезис:

```
[T001] ТИП | Спикер — Самодостаточная формулировка (понятна без контекста)
```

Типы: ФАКТ, CLAIM, ВОПРОС, ИДЕЯ, РЕШЕНИЕ, ПРОБЛЕМА

Правила:
- Полнота: извлекай ВСЁ, даже мелкие реплики со смыслом
- Самодостаточность: каждый тезис понятен БЕЗ транскрипта
- Атрибуция: всегда указывай спикера
- Без интерпретации: записывай что сказано, не что "имелось в виду"

### 4. Дерево дискуссии

Построй граф связей между тезисами в псевдографике (ASCII):

```
┌─────────────────────────────┐
│ T001 ФАКТ | Спикер          │
│ [формулировка тезиса]       │
└──────────────┬──────────────┘
               │ причина
               ▼
┌──────────────────────────────┐    ──✕──    ┌──────────────────────────────┐
│ T002 CLAIM | Спикер          │◄───────────►│ T003 CLAIM | Спикер          │
│ [формулировка]               │  возражение │ [формулировка]               │
└──────────────┬───────────────┘             └──────────────────────────────┘
               │ развитие
               ▼
┌──────────────────────────────┐
│ T005 РЕШЕНИЕ | Спикер        │
│ [формулировка]               │
└──────────────────────────────┘
```

Символы связей:
- `───▶` причина-следствие
- `══▶` поддержка
- `──✕──` возражение/конфликт
- `···▶` уточнение
- `--▷` развитие
- `?──▶` вопрос-ответ

Группируй по кластерам с рамками:

```
╔══════════════════════════════════════╗
║  КЛАСТЕР: [Название темы]           ║
╠══════════════════════════════════════╣
║  [узлы и связи внутри кластера]     ║
╚══════════════════════════════════════╝
```

Обязательно подсвети:
- Конфликты (пары несовместимых тезисов)
- Причинно-следственные цепочки (A привело к B привело к C)
- Развилки (где обсуждались альтернативы)

Для больших графов (30+ тезисов) — разбей на несколько кластеров.

### 5. Нарратив дискуссии

Для каждой ветки — связное описание со ссылками на [Txxx]:

> Дискуссия началась с [T001], когда Speaker отметил... Это вызвало [T003]... В итоге пришли к [T012].

### 6. Выходные списки

#### ИНСАЙТЫ
Новые понимания, неочевидные связи, "ага-моменты". То, что участники поняли в процессе обсуждения.

#### ПРИЁМЫ МЕНТОРА/ТРЕКЕРА
Какие техники использовал ведущий/трекер/ментор (если есть):
- Сократический метод (задавать вопросы)
- Рефрейминг
- Приоритезация
- Декомпозиция
- Супервизия
- Анализ рисков
- Анализ обратной связи
- Делегирование
- Другие из списка СКИЛЫ

#### ИДЕИ ДЛЯ СТАТЕЙ
Темы, тезисы или заготовки для блога/канала, вытекающие из обсуждения.
Формат каждой идеи:
- **Рабочее название** — 1-2 предложения о чём писать + ключевые тезисы из транскрипта

#### КАНДИДАТЫ В TODO
Задачи, которые явно или неявно возникли в ходе встречи. Формулируй по Дорофееву:
1. Начинается с глагола
2. Первый конкретный шаг
3. Понятный результат
4. До 30 минут

#### ДОБЫЧА (по справочнику тегов)

Распредели найденное по категориям:

**Энергия:**
- Творческие находки: идеи, инсайт, озарение, гипотеза, дизайн, сюжет
- Новые возможности: новые поводы, продажа, прототип, модель, методика, заготовка
- Социальное: новые друзья, репутация, хороший вопрос
- Личное: упражнение, прием, корм

**Целевые:**
- Бизнес: клиент, канал продаж, прибыль, спайс, блокер снял
- Развитие: цель, кейс, лучшая практика, новый процесс, интересная задача
- Коллаборация: взаимные интересы, союзник, партнер, материал для вдохновения
- Знание: целевое знание, факты, фреймворк

**Проблемы:**
- Процессные: накладка, не вывез, прокрастинация
- Состояния: надо понять, фейл

Пиши только то, что реально прозвучало. Не выдумывай.
