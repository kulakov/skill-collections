# Как пользоваться Strategy Safari

Strategy Safari — каталог 60 валидированных методик стратегического анализа, организованных по 10 школам Минцберга. Каждая методика — с каноническим протоколом из первоисточника, без произвольных пересказов.

Существует в двух формах:

- **Каталог-витрина** на [method.lance.ru](https://method.lance.ru/methodologies) — справочник для всех, фильтры по школам и фасетам, читать без установки.
- **Скилл для Claude Code** — диалоговый ассистент, который проводит по каноническому протоколу, делает gate на стратегическую триаду, комбинирует методики в цепочки.

Эта вики — два раздела. **Часть A** — для всех, без Claude Code. **Часть B** — для тех у кого Claude Code установлен.

## Принцип: 10 школ Минцберга

Генри Минцберг в книге *Strategy Safari* (1998) показал: стратегическая мысль — это десять разных школ, каждая описывает одну часть слона. Ни одна не полная.

| # | Школа | Стратегия как… | Ключевая идея |
|---|-------|----------------|----------------|
| 1 | Дизайна | концепция | согласование внутреннего и внешнего (SWOT) |
| 2 | Планирования | формальный процесс | разложить на шаги, посчитать, спланировать |
| 3 | Позиционирования | анализ | выбрать защитимую позицию в отрасли (Porter, BCG) |
| 4 | Предпринимательства | видение | стратегия в голове лидера |
| 5 | Познания | ментальный процесс | как мы думаем о стратегии |
| 6 | Обучения | эмерджентный процесс | стратегия рождается из опыта |
| 7 | Власти | переговоры | стратегия как результат торга стейкхолдеров |
| 8 | Культуры | коллективный процесс | стратегия из общих ценностей и ресурсов (RBV) |
| 9 | Внешней среды | реакция | организация — щепка в потоке среды |
| 10 | Конфигурации | трансформация | стратегия как смена устойчивых состояний |

**Задача каталога — подобрать правильную комбинацию методик для конкретной ситуации.** Не «есть один правильный фреймворк», а «вот пять линз, посмотри через каждую и сравни картину».

## Стратегическая триада (универсальный gate)

Это работает с Claude Code и без него. **Перед любым стратегическим анализом** проверь три условия:

| # | Условие | Проверочный вопрос | Если отсутствует |
|---|---------|-------------------|------------------|
| 1 | **Субъект стратегии** | Кто принимает стратегические решения? Есть ли лицо или группа с полномочиями и ресурсами? | Нет стратега — нет стратегии. Будут только рекомендации в воздух |
| 2 | **Объект стратегии** | Что именно мы стратегически ведём? Где границы? (компания / продукт / рынок / подразделение) | Размытый объект — невозможно сфокусировать анализ |
| 3 | **Намерение** | Чего субъект хочет от/с объектом? Есть ли направление, пусть неточное? | Нет намерения — нет стратегии, только описание состояния |

**Если хотя бы одно отсутствует — не пытайся «натянуть» стратегию.** Сначала разберись с тем, чего не хватает. Это уберегает от часов работы над фальшивым ТЗ.

# Часть A — Без Claude Code

## A1. Зачем тебе каталог

| Роль | Что даёт каталог |
|------|------------------|
| **Консультант** | Быстро найти подходящий фреймворк под клиента, увидеть ограничения метода |
| **Основатель** | Подготовиться к стратегической сессии, не изобретать велосипед |
| **Трекер стартап-команд** | Иметь под рукой 60 методик с протоколами для разных типов запросов |
| **Студент / самообразование** | Структурированный обзор стратегического мышления, не расплываясь в книгах |
| **Менеджер продукта** | Выбрать линзу для конкретного решения (позиция, ресурсы, рост) |

Каталог НЕ учит думать стратегически — он даёт **инструменты, проверенные первоисточниками**, и помогает не использовать их вслепую.

## A2. Как выбрать методику

Алгоритм работает универсально, но особенно полезен в первый раз:

**Шаг 1. Триада-gate.** Если нет субъекта/объекта/намерения — стоп.

**Шаг 2. Какая часть слона тебе нужна?**

- Нужно выбрать позицию на рынке → школа Позиционирования
- Понять внутренние ресурсы и ключевые компетенции → Культуры (RBV) и Обучения
- Спланировать рост по шагам → Планирования
- Создать что-то новое, нет данных → Предпринимательства
- Провести изменения, кризис → Конфигурации
- Много стейкхолдеров с конфликтующими интересами → Власти
- Высокая неопределённость → Планирования (сценарии) + Обучения (эмерджентная)
- Команда не видит очевидного → Познания (когнитивный аудит)
- Первый стратегический анализ → Дизайна (SWOT)

**Шаг 3. Возьми 2–3 методики из разных школ.** Одна методика = одна линза. Если использовать только одну — ослепнешь на остальное.

**Шаг 4. Найди карточку на сайте.** На [method.lance.ru/methodologies](https://method.lance.ru/methodologies) есть фильтры по школе, фокусу и use case.

## A4. Типичные комбинации

Цепочки из 4–6 методик из разных школ под типовой запрос.

### Полный стратегический аудит
1. **SWOT** (Дизайн) — первичная карта
2. **5 сил Портера** (Позиционирование) — отраслевой контекст
3. **Value Chain** (Позиционирование) — где создаём ценность
4. **VRIN** (Культура) — что неимитируемо
5. **Stakeholder Analysis** (Власть) — кто влияет на решения
6. **Scenario Planning** (Планирование) — при высокой неопределённости

### Стартап ищет стратегию
1. **Артикуляция видения** (Предпринимательство) — куда идём
2. **5 сил** (Позиционирование) — привлекательна ли отрасль
3. **Core Competence** (Обучение) — что умеем уникального
4. **Emergent Strategy** (Обучение) — что уже работает без плана

### Кризис и трансформация
1. **Miles & Snow** (Конфигурация) — кто мы сейчас
2. **Greiner lifecycle** (Конфигурация) — на какой стадии
3. **Cognitive Bias Audit** (Познание) — что мешает видеть
4. **Turnaround protocol** (Конфигурация) — план действий

### Конфликт между основателями
1. **Stakeholder Analysis** (Власть) — карта позиций
2. **Cognitive Mapping** (Познание) — у каждого свой ментальный образ
3. **Coalition Building** (Власть) — где возможны альянсы

## A5. Как читать карточку методики

Каждая карточка устроена одинаково:

| Поле | Что там |
|------|---------|
| **Source** | Кто, в какой работе, в каком году. Это позволяет проверить и углубиться |
| **Canonical Protocol** | Шаги из первоисточника, не пересказ |
| **Validation Context** | Где методика валидирована (отрасли, размер компаний) |
| **Known Limitations** | Известные слабости — НЕ скрыты |
| **When to combine** | С какими методиками из других школ работает |

Если в карточке протокол не из первоисточника — карточка плохая. Канонические шаги — не педантизм, а защита от «брейншторминга под вкусом методологии».

## A6. Шесть принципов качественного анализа

1. **Только канонический протокол.** Шаги — из первоисточника. Произвольные пересказы дают анализ, который выглядит правдоподобно, но пуст.
2. **Называй источник.** При проведении методики упоминай автора, работу, год. Это позволяет проверить.
3. **Указывай ограничения.** Каждая методика имеет известные слабости. Не скрывай — предупреждай.
4. **Предлагай дополнение.** После одной методики — скажи, что она НЕ покрывает и какая методика из другой школы закроет пробел.
5. **Диалоговый формат.** Не выкладывай весь анализ сразу. Задавай вопросы по протоколу, получай ответы, двигайся дальше.
6. **Комбинируй осознанно.** Методики из разных школ могут противоречить друг другу — это нормально и ценно. Покажи противоречие, не сглаживай.

# Часть B — С Claude Code

## B1. Что добавляет скилл

Каталог-витрина — это справочник. **Скилл** — это диалоговый ассистент, который:

1. **Делает gate на триаду автоматически.** Не пускает в анализ, если нет субъекта/объекта/намерения.
2. **Маршрутизирует по 10 школам** через диагностические вопросы про контекст и тип задачи.
3. **Ведёт по каноническому протоколу диалогом** — задаёт вопросы методики, не выкладывает всё сразу.
4. **Комбинирует методики в цепочки** — после одной предлагает следующую из другой школы, чтобы закрыть пробел.
5. **Указывает ограничения** в конце каждой методики, не дав пользователю переоценить вывод.

Это не «спросил Claude про SWOT — он расскажет про SWOT». Это полноценный фасилитатор стратегической сессии.

```bash
# 1. Поставить Claude Code (если ещё нет)
# https://docs.claude.com/claude-code

# 2. Положить файлы скилла
mkdir -p ~/.claude/skills/strategy-safari/methods

# 3. Скопировать SKILL.md и methods/*.md из репозитория
# (или клонировать из приватного репо kulakov)

# 4. Перезапустить Claude Code — скилл подхватится автоматически
```

Структура папки:

```

├── SKILL.md           # оркестратор: триада-gate, 3 режима, навигатор
└── methods/           # 15 готовых методик (из 60)
    ├── 01-SWOT-Analysis.md
    ├── 02-Porter-Five-Forces.md
    ├── ...
    └── 14-Greiner-Growth-Model.md
```

Скилл триггерится автоматически по фразам в `description:` (см. ниже).

## B3. Триггеры

Скилл активируется когда говоришь Claude Code:

**Общие**
- «помоги со стратегией [X]»
- «стратегический анализ для [Y]»
- «какую стратегию выбрать»
- «есть ли у нас стратегия»
- «strategy safari», «школы стратегии», «Минцберг»

**Конкретные методики**
- «проведи 5 сил Портера для [X]»
- «SWOT для нашего стартапа»
- «Value Chain для [Y]»
- «BCG матрица для портфеля [Z]»
- «стейкхолдер-анализ»
- «сценарное планирование»

**Диагностические**
- «на какой мы стадии» → Greiner
- «кто мы — defender или prospector» → Miles & Snow
- «что у нас неимитируемо» → VRIN

**НЕ триггерится для**: анализа конкурентов без стратегического контекста, метрик стартапа, ценообразования, финмоделей, свободного размышления, критики идеи, трендов. Для них есть другие скиллы.

## B4. Три режима работы

### Режим 1. Диагностика (по умолчанию)

Ты приходишь с ситуацией («помоги разобраться со стратегией для [X]») — скилл задаёт диагностические вопросы:

- Размер компании? Стадия? Отрасль?
- Что нужно — выбрать позицию / понять ресурсы / спланировать рост / провести изменения / разобраться со стейкхолдерами / снять неопределённость / увидеть слепые пятна?
- Есть ли существующая стратегия? Что не работает?

После диагностики предлагает 2–3 методики из подходящих школ, объясняет почему именно они, и предлагает последовательность.

### Режим 2. Прямой вызов методики

Ты называешь конкретную методику («проведи 5 сил Портера для [X]») — скилл:

1. Загружает файл методики из `methods/`
2. Проводит по каноническому протоколу из первоисточника, диалогом
3. В конце называет ограничения и предлагает методику из другой школы, чтобы закрыть пробел

### Режим 3. Комбинирование

Нужен комплексный анализ — скилл собирает цепочку методик из разных школ. Цепочки из части A4 — типичные шаблоны.

## B5. Три примера сессии

### Пример 1. Стратегическая сессия для стартапа

```
Ты: помоги со стратегией для нового SaaS — у меня есть прототип, ищу куда копать
Claude (триада-gate):
  1. Кто принимает стратегические решения? (ты один или есть кофаундер?)
  2. Что именно объект — продукт, компания, рынок?
  3. Куда хочешь — выручка, фандрейз, продажа?
Ты: один, объект = продукт + компания, цель — найти PMF и закрыть seed
Claude (диагностика):
  Стадия запуска, нужно понять отрасль и ключевые компетенции.
  Предлагаю цепочку: 5 сил → Core Competence → Emergent Strategy.
  Начнём с 5 сил — ответь на 8 вопросов про отрасль…
```

### Пример 2. Аудит стратегии клиента (как трекер)

```
Ты: команда [X] говорит что у них «нет стратегии», провести аудит
Claude: триада-gate — субъект, объект, намерение?
Ты: основатель, продукт, расти 3x за год
Claude: рекомендую полный аудит — SWOT → 5 сил → VRIN → Stakeholders.
  Начнём со SWOT, у тебя есть данные про сильные/слабые стороны команды?
[…]
Claude (после SWOT): SWOT не показывает динамику отрасли. Переходим к 5 силам.
```

### Пример 3. Личное решение

```
Ты: я думаю запускать новый бизнес — есть ли у меня шансы
Claude: триада — субъект ты, объект какая идея, намерение какое?
Ты: я, идея — AI-инструмент для [X], хочу sustainable revenue
Claude: для опции «индивидуальная стратегия» нужны три линзы:
  Core Competence (что я умею уникального),
  5 сил (отрасль выдержит),
  Emergent Strategy (что у меня уже работает без плана).
  Начнём с Core Competence…
```

## B6. Как добавить новую методику

1. Создай файл `~/.claude/skills/strategy-safari/methods/NN-Name.md`
2. Структура файла:

```markdown
# [Название методики]

## Original Source
Автор, работа, год, страницы — обязательно.

## Canonical Protocol
Шаги из первоисточника, пронумерованы. Не пересказ.

## Fit Check
2–3 вопроса yes/no — проверить, подходит ли методика к ситуации.

## Validation Context
Где валидирована — отрасли, размеры компаний, ограничения применимости.

## Known Limitations
Известные слабости. Что НЕ покрывает.

## Combines With
Какие методики из других школ закрывают пробелы этой.
```
