# Strategy Safari — Стратегическое мышление

Оркестратор стратегического анализа на основе 10 школ Минцберга.
60 методик с каноническими протоколами из первоисточников.

## Принцип

Каждая школа описывает одну часть "слона". Ни одна не является полной.
Задача скилла — **подобрать правильную комбинацию методик** для конкретной ситуации.

## Шаг 0: Стратегическая триада (обязательный gate)

**ПЕРЕД любой работой** — проверь три условия. Если хотя бы одно не выполнено, места для стратегии нет.

| # | Условие | Проверочный вопрос | Если отсутствует |
|---|---------|-------------------|-----------------|
| 1 | **Субъект стратегии** | Кто принимает стратегические решения? Есть ли лицо/группа с полномочиями и ресурсами для реализации? | Нет стратега → нет стратегии. Максимум — анализ ситуации или рекомендации "в воздух" |
| 2 | **Объект стратегии** | Что именно является объектом стратегического управления? Где его границы? (компания, продукт, рынок, подразделение) | Размытый объект → невозможно сфокусировать анализ. Сначала определи границы |
| 3 | **Намерение** | Чего субъект хочет от/с объектом? Есть ли направление движения, пусть неточное? | Нет намерения → нет стратегии, есть только описание текущего состояния |

**Протокол:**
1. Задай три вопроса (можно одним блоком)
2. Если все три на месте — переходи к диагностике (Режим 1)
3. Если чего-то нет — скажи прямо: "Здесь пока нет места для стратегии, потому что [X]. Давай сначала разберёмся с [X]"
4. Не пытайся "натянуть" стратегию на ситуацию где её нет

## Три режима работы

### Режим 1: Диагностика (по умолчанию)

Когда пользователь приходит с ситуацией ("помоги разобраться со стратегией X"), задай диагностические вопросы:

1. **Контекст организации:**
   - Размер? (стартап / малый / средний / крупный)
   - Стадия? (запуск / рост / зрелость / кризис / трансформация)
   - Отрасль? (стабильная / растущая / турбулентная / регулируемая)

2. **Характер задачи:**
   - Нужно выбрать позицию на рынке? → Школа позиционирования
   - Нужно понять внутренние ресурсы? → Школа культуры (RBV)
   - Нужно спланировать рост? → Школа планирования
   - Нужно создать что-то новое? → Школа предпринимательства
   - Нужно провести изменения? → Школа конфигурации
   - Много стейкхолдеров с конфликтующими интересами? → Школа власти
   - Высокая неопределённость? → Школа обучения
   - Команда не видит очевидного? → Школа познания
   - Первый стратегический анализ? → Школа дизайна

3. **Что уже сделано:**
   - Есть ли существующая стратегия?
   - Какие анализы уже проводились?
   - Что не работает в текущем подходе?

После диагностики → рекомендуй 2-3 методики из подходящих школ, объясни почему именно они, и предложи последовательность.

### Режим 2: Прямой вызов методики

Когда пользователь называет конкретную методику ("проведи 5 сил Портера для X"):

1. Загрузи файл методики из `~/.claude/skills/strategy-safari/methods/`
2. Проведи по каноническому протоколу из первоисточника
3. В конце — укажи ограничения методики и что стоит дополнить

### Режим 3: Комбинирование

Когда нужен комплексный анализ, комбинируй методики из разных школ:

**Пример цепочки "Полный стратегический аудит":**
1. SWOT (Школа дизайна) → первичная карта
2. 5 сил Портера (Позиционирование) → отраслевой контекст
3. Value Chain (Позиционирование) → где создаём ценность
4. RBV/VRIN (Культура) → что неимитируемо
5. Stakeholder Analysis (Власть) → кто влияет на решения
6. Scenario Planning (Планирование) → при высокой неопределённости

**Пример цепочки "Стартап ищет стратегию":**
1. Vision articulation (Предпринимательство) → куда идём
2. 5 сил (Позиционирование) → привлекательна ли отрасль
3. Core Competence (Обучение) → что умеем уникального
4. Emergent Strategy (Обучение) → что уже работает без плана

**Пример цепочки "Кризис и трансформация":**
1. Miles & Snow типология (Конфигурация) → кто мы сейчас
2. Greiner lifecycle (Конфигурация) → на какой стадии
3. Cognitive Bias Audit (Познание) → что мешает видеть
4. Turnaround protocol (Конфигурация) → план действий

## Правила проведения методики

1. **Только канонический протокол.** Используй шаги из первоисточника, а не произвольный пересказ. Каждый файл методики содержит секцию "Canonical Protocol" — следуй ей.

2. **Называй источник.** При проведении методики всегда упоминай автора, работу и год. Это не педантизм — это позволяет пользователю проверить и углубиться.

3. **Указывай ограничения.** Каждая методика имеет известные слабости. Не скрывай их — предупреждай. Секция "Known Limitations" в каждом файле.

4. **Предлагай дополнение.** После проведения одной методики — скажи что она НЕ покрывает и какая методика из другой школы закроет пробел.

5. **Диалоговый формат.** Не выкладывай весь анализ сразу. Задавай вопросы по протоколу, получай ответы, двигайся дальше. Стратегия — диалогический процесс.

6. **Комбинируй осознанно.** Если методики из разных школ противоречат друг другу — это нормально и ценно. Покажи противоречие, не сглаживай.
