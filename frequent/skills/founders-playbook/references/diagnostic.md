# Диагностика стадии команды

Этот файл — обязательный gate перед любыми упражнениями. Без правильно определённой стадии скилл бесполезен.

## Базовая логика

| Что у команды есть | Стадия |
|--------------------|--------|
| Только идея + предположения о пользователях | Idea (ранняя) |
| Идея + 10+ интервью с целевыми пользователями, нет билда | Idea (поздняя) |
| Прототип/MVP в руках реальных пользователей, но retention/PMF не подтверждены | MVP |
| PMF подтверждён (retention/revenue/referral от identifiable сегмента) | Launch |
| Воспроизводимый channel-driven рост, операции без фаундера-bottleneck | Scale |

## Блок диагностических вопросов

Задавай по одному. Не вываливай списком.

### Вопрос 1 — про проблему

> «Можете назвать конкретный сегмент людей, у которых эта проблема, как часто они с ней сталкиваются, насколько остро, и что они делают сейчас?»

- Не может назвать → **Idea (ранняя)**
- Может, но это observation не tested hypothesis («все мои знакомые жалуются на X») → **Idea (ранняя)**
- Может с цифрами из интервью («50% mid-market finance managers тратят 4+ часов/нед на X, делают это в Excel») → переходим к Вопросу 2

### Вопрос 2 — про билд

> «Есть ли работающий продукт, которым пользуются реальные люди вне ближнего круга (не друзья, не портфельные компании инвестора, не Hacker News splash)?»

- Нет → **Idea**
- Есть, но пока 1-5 пользователей и непонятно вернутся ли → **MVP (ранняя)**
- Есть, 20+ пользователей, есть данные по поведению → переходим к Вопросу 3

### Вопрос 3 — про PMF

> «Какой процент активных пользователей сказал бы 'очень разочарован', если бы продукта больше не было? (Sean Ellis test)»

- Не знают / не проводили → **MVP**, нужен measurement framework
- <40% → **MVP**, продукт ещё не PMF
- 40%+ И есть retention curve которая стабилизируется (не уходит в ноль) → переходим к Вопросу 4

Дополнительный effort test:
> «Что произойдёт с retention, если вы перестанете лично пушить пользователей (outreach, инсентивы, follow-up)? Продукт начинает работать сам?»

- Нет, всё держится на героизме фаундера → **MVP**, не PMF
- Да, есть organic pull → PMF подтверждён, переходим дальше

### Вопрос 4 — про рост

> «Знаете ли вы CAC, LTV, payback period? Может ли команда воспроизводимо приводить пользователей через конкретные каналы?»

- Нет / органика из ничего → **Launch (ранняя)**, нужен measurement + GTM-функция
- Есть unit economics, 1-2 канала работают → переходим к Вопросу 5

### Вопрос 5 — про operations

> «Если фаундер уезжает на неделю — что встаёт? Какие решения проходят через него ежедневно?»

- Встаёт всё → **Launch**, фаундер-bottleneck не убран
- Встают только стратегические решения, операции идут → **Scale**

## Красные флаги (намекают на самообман в диагностике)

| Сигнал | Что это значит |
|--------|----------------|
| «У нас уже PMF, потому что Hacker News нас залайкал» | Ephemeral traction. На самом деле — **MVP** |
| «У нас 1000 sign-ups» | Sign-ups без активации/retention — **false positive**. **MVP** |
| «Мы на Launch, потому что построили продукт» | Билд ≠ стадия. **Если retention не подтверждён — MVP** |
| «Мы на Scale, потому что хотим хайрить» | Желание не равно готовности. Прогон через Launch exit criteria |
| «Мы на Idea, но уже пишем код» | Можно. Главное чтобы билд работал на валидацию, а не на «у меня же есть прототип» |

## Что делать, если стадия неоднозначна

1. **По умолчанию — более ранняя стадия.** Лучше переделать exit criteria предыдущей чем строить на песке.
2. **Если команда настаивает на более поздней** — попроси доказательства exit criteria стадии, к которой они себя относят. Если не могут привести — определяем стадию по тем criteria, которые реально выполнены.
3. **Если команда «между» стадиями** (например, Idea закрыта на 80%) — возьмите оставшиеся upcoming exit criteria и сделайте их явными tasks в текущей сессии.

## После диагностики

1. Назови команде стадию вслух
2. Прочитай её цель (одна фраза из stage-файла)
3. Прочитай exit criteria
4. Спроси: «согласны что вы здесь?»
5. Переходи к упражнениям только после согласия

Если команда возражает — не спорь, попроси argumets. Возможно ты пропустил данные. Возможно команда защищает self-image. Оба варианта надо разрешить до упражнений.
