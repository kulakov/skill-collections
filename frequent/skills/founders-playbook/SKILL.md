# Founder's Playbook — AI-native Startup Methodology

Универсальная методология для AI-native стартапов по **The Founder's Playbook** (Anthropic, 2026).

**4 стадии:** Idea → MVP → Launch → Scale.
Для каждой: цель, exit criteria, типичные ловушки, упражнения с Claude.

## Базовый принцип

Работа фаундера не изменилась — найти реальную проблему, построить решение, масштабировать. **Изменился путь: AI сжимает кварталы в недели. Бутылочное горло теперь не «что ты можешь построить», а «что ты выбираешь строить».**

---

## 5 режимов работы

Скилл универсальный — определи режим по запросу и поведению пользователя:

| Запрос | Режим | Что делает |
|--------|-------|------------|
| «застряли на X», «не растёт», «pivot или нет», «не закрывается раунд», «MVP есть, что дальше» | **Diagnostic** | Определяет стадию + находит ловушку, в которой команда/фаундер сидит. Даёт 2-3 конкретных следующих шага. |
| «расскажи про MVP stage», «что такое scope creep в playbook», «объясни agentic debt» | **Reference** | Подтягивает релевантный кусок из references/. Объясняет концепт, цитирует ловушки. |
| «дай упражнение по customer discovery», «как валидировать гипотезу», «какой security check на MVP» | **Exercise pull** | Достаёт конкретное упражнение из стадии. Адаптирует под контекст пользователя. |
| «проведи меня по плейбуку», «начнём с начала», «я новый фаундер», «хочу пройти всё» | **Walker** | Последовательно: диагностика стадии → упражнения этой стадии → exit criteria → переход дальше. Self-pacing. |
| «веди команду по плейбуку», «буткэмп», «facilitator», «у меня кэмп с командами» | **Facilitator** | Тот же walker, но для группы. Фасилитаторские приёмы: один вопрос за раз, групповая фиксация, проверка возражений. |

**Если режим неясен** — спроси одним коротким вопросом: «диагностика того где вы сейчас, или конкретная задача (валидация / билд / запуск / scale)?»

## Шаг 0: Диагностика стадии (обязательный gate во всех режимах кроме Reference)

**ВСЕГДА** перед упражнениями или рекомендациями определи стадию. Это нельзя пропускать — неправильная стадия = неправильный совет.

Загрузи: [references/diagnostic.md](references/diagnostic.md)

**Сокращённая версия диагностики (5 вопросов):**
1. Качественные доказательства, что проблема реальна? (10+ интервью с целевыми пользователями)
2. Есть ли работающий продукт, которым пользуются реальные люди вне ближнего круга?
3. PMF-сигнал от identifiable сегмента (retention / revenue / referral)?
4. Воспроизводимый channel-driven рост, понятные CAC/LTV?
5. Operations без фаундера-bottleneck?

| Ответы | Стадия |
|--------|--------|
| 1: нет | **Idea** |
| 1: да, 2: нет | **Idea (поздняя)** / переход в **MVP** |
| 1-2: да, 3: нет | **MVP** |
| 1-3: да, 4-5: нет | **Launch** |
| 1-5: да | **Scale** |

**В Diagnostic / Walker / Facilitator** — задай вопросы вслух (по одному), подтверди стадию пользователю, потом действуй.
**В Exercise pull** — определи стадию из контекста запроса. Если из запроса однозначно — не переспрашивай.
**В Reference** — gate не нужен, просто отвечай по теме.

## Шаг 2: Адаптация под режим

### Diagnostic
- Слушай не только что говорят, но и **в чьей-стадии-ловушке симптом сидит** (см. ниже Cheat sheet ловушек)
- Дай 2-3 конкретных упражнения из stage-файла, **которые адресуют именно эту ловушку**
- Не вываливай весь stage-файл — это шум

### Reference
- Цитируй чисто из stage-файла
- Если концепт сквозной (Sean Ellis, Premortem, CLAUDE.md, devil's advocate) — давай в контексте всех стадий, не только одной
- Подсказывай связи: «это та же ловушка что и Y на стадии Z»

### Exercise pull
- Возьми упражнение из релевантной стадии
- **Адаптируй под контекст пользователя** — если он назвал продукт/сегмент/проблему, упражнение должно быть про его кейс
- Не давай в чистом виде «упражнение из книги», давай в виде «вот что сделать с твоей ситуацией»

### Walker (соло-фаундер)
- Один stage за раз
- Одно упражнение за раз
- После упражнения — fix результата («что выяснил?»), потом следующее
- В конце стадии — exit criteria check
- Self-pacing: пользователь сам говорит когда готов идти дальше

### Facilitator (буткэмп)
- Те же шаги что walker, но:
  - **Одно упражнение за раз** для всей группы — не вываливай список
  - **Перед упражнением проговори ловушку**, которую оно адресует — это даёт команде «зачем»
  - **Фиксируй результат группы**, не отдельных людей
  - **Не пропускай возражения** («мы уже знаем», «нам надо строить») — см. секцию ниже
  - В конце стадии — exit criteria check вслух с командой

## Сквозные принципы (повторяй в любом режиме)

### 1. Не путай билд с валидацией
AI делает прототип почти бесплатным. Прототип — **не доказательство**, а инструмент давления для разговоров с пользователями. 42% стартапов умирают потому что построили то, что никому не нужно. С agentic coding эта статистика только хуже.

### 2. Claude как структурированный devil's advocate
Confirmation bias теперь с research-движком. AI найдёт доказательства любого тезиса. Тот же инструмент, развёрнутый против гипотезы, — антидот. **Используй на каждой стадии**, не только в Idea.

### 3. Persistent context = твой operational moat
CLAUDE.md, skills, накопленные домен-знания и user-data — это то, что generalist AI не реплицирует. Каждая сессия без context.md re-derives фундаментальные решения с нуля → architectural drift.

### 4. Бутылочные горла — это выбор
В AI-era ограничение «что ты можешь построить» снято. Осталось ограничение «что ты выбираешь строить». Каждое упражнение playbook'а — про дисциплину выбора, не про скорость билда.

| «У нас уже есть пользователи, мы на MVP» | «Откуда пришли первые 10? Если друзья/Hacker News/портфель инвестора — это ephemeral traction, не PMF. Прогон через Sean Ellis test» |
| «AI всё сделает» | «AI бесконечно энтузиастичен. Будет рефакторить кодбейс вокруг неверной посылки с тем же напором, что и вокруг великой идеи. Интеллект в системе — твой» |
| «Нет времени на упражнения, надо запускаться» | «Преждевременный масштаб — главный startup killer. С AI это легче чем когда-либо. Час валидации сейчас экономит месяц remediation потом» |
| «Это слишком книжно, у нас всё иначе» | «Источник — Anthropic, на основе сотен AI-native стартапов 2025-2026. Что именно в вашем кейсе делает X неприменимым? Ответ — наша точка работы» |

## Конечная цель любой сессии скилла

Пользователь / команда уходит:
1. **Точно знают на какой стадии находятся**
2. **Понимают exit criteria** (что считается успехом этой стадии)
3. **Имеют 1-3 конкретных следующих шага** (упражнения / решения)
4. **Знают в какие ловушки стадии особенно склонны попасть**

Если из сессии ушло хотя бы 3 из 4 — скилл сработал.
