# Разработка

Скиллы для кодинга: спека (SDD/ITD), архитектура, ревью, MCP/агенты, автоматизация, фронтенд, песочница.

**37 скиллов.** Положи папки/файлы в `~/.claude/skills/` и `~/.claude/commands/`.

## Состав

- **ai-agents-architect** — Архитектура автономных AI-агентов: tool use, память, планирование, оркестрация
- **ai-product** — Разработка AI-продуктов: LLM-интеграция, RAG, prompt engineering, AI UX, оптимизация косто
- **ai-wrapper-product** — Продукты-обёртки над AI API (OpenAI, Anthropic)
- **autonomous-researcher** — Автономный AI-исследователь с параллельными агентами
- **docker-sandbox** — Запуск Claude Code в Docker Sandbox без подтверждений
- **frontend-design** — Выразительные production-grade фронтенд-интерфейсы высокого качества
- **l-d-agent-sdk** — Скаффолд нового проекта на Claude Agent SDK
- **l-d-cli** — Создание composable CLI-инструмента из API docs, OpenAPI, curl, SDK или скрипта
- **l-d-local** — Запуск Claude Code с локальной моделью через Ollama
- **l-d-vibe-setup** — Управление сетапом вайб-дизайна JetStyle: статус инструментов, роадмап, scaffold проектов,
- **l-nic** — NIC Method — структурированная фасилитация проекта (Brief -> Challenge -> Debrief -> Plan 
- **l-x-frontend** — Создание выразительных фронтенд-интерфейсов с высоким качеством дизайна
- **mcp-builder** — Создание MCP-серверов (Model Context Protocol)
- **multi-agent-patterns** — Паттерны мульти-агентных систем: оркестратор, peer-to-peer, иерархия
- **n8n-mcp-tools-expert** — Эксперт по n8n-mcp MCP инструментам
- **owasp-security** — OWASP security review
- **port-doctor** — Управление локальными сервисами и портами
- **prompt-engineering-patterns** — Продвинутые техники prompt engineering для LLM
- **repo-task-proof-loop** — Верифицируемый цикл разработки для сложных задач: freeze spec → build → evidence → verify 
- **schema** — Schema.org разметка и структурированные данные на сайте
- **sdd-spec-design** — Создание технического дизайна для SDD-спецификации
- **sdd-spec-impl** — Выполнение задач спецификации по TDD-методологии
- **sdd-spec-init** — Инициализация новой SDD-спецификации с описанием проекта
- **sdd-spec-requirements** — Генерация требований для SDD-спецификации
- **sdd-spec-status** — Статус и прогресс SDD-спецификации
- **sdd-spec-tasks** — Генерация задач имплементации для SDD-спецификации
- **sdd-steering** — Управление steering-файлами как persistent project knowledge
- **sdd-validate-design** — Интерактивная проверка качества технического дизайна
- **sdd-validate-gap** — Анализ разрыва между требованиями и существующим кодом
- **sdd-validate-impl** — Валидация имплементации против требований, дизайна и задач
- **security-reviewer** — Security reviewer subagent
- **senior-review** — Senior Engineer Review — полный ревью кода с паузами между секциями
- **tanenbaum** — Архитектура распределённых систем по Таненбауму. Используй при задачах об интеграциях межд
- **telegram-bot-builder** — Создание Telegram-ботов: от простой автоматизации до AI-ботов
- **telegram-mini-app** — Создание Telegram Mini Apps (TWA)
- **workflow-automation** — Автоматизация воркфлоу с кодом: n8n, Temporal, Inngest, durable execution
- **zapier-make-patterns** — No-code автоматизация через Zapier и Make
