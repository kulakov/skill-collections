## ФАЗА 1: ДЕКОМПОЗИЦИЯ

**Роль:** Principal Investigator — разбей исследовательский вопрос на тестируемые гипотезы.

### Формат гипотезы

```markdown
## Гипотеза H1: [Название]

**Вопрос:** Что именно проверяем?

**Метод:** Как будем проверять?
- Эксперимент / код / веб-поиск / анализ данных

**Ожидаемый результат:** Что подтвердит/опровергнет гипотезу?

**Зависимости:** Требуются ли результаты других гипотез?
```

### Правила декомпозиции

1. **3-5 гипотез** — не больше, иначе расфокус
2. **Независимость** — большинство можно проверять параллельно
3. **Тестируемость** — каждая гипотеза имеет чёткий критерий успеха
4. **Полнота** — вместе гипотезы покрывают исходный вопрос

### Спроси пользователя

> Я разбил задачу на N гипотез:
> - H1: ...
> - H2: ...
> - H3: ...
>
> Запустить агентов для проверки? [Да / Изменить / Добавить]

## ФАЗА 2: ЭКСПЕРИМЕНТЫ (ПАРАЛЛЕЛЬНЫЕ АГЕНТЫ)

**КРИТИЧЕСКИ ВАЖНО:** Используй `Task` tool с `subagent_type: "general-purpose"` для запуска агентов.

### Как запускать агентов

Для каждой независимой гипотезы — отдельный Task в **одном сообщении** (параллельно):

```
Task tool:
  subagent_type: "general-purpose"
  description: "Проверка гипотезы H1"
  prompt: |
    Ты — автономный исследователь. Твоя задача:

    ## Гипотеза
    [Полное описание гипотезы]

    ## Инструкции
    1. Собери релевантную информацию (WebSearch, WebFetch)
    2. Если нужен код — напиши и запусти (Bash)
    3. Проанализируй результаты
    4. Сформулируй вывод: ПОДТВЕРЖДЕНА / ОПРОВЕРГНУТА / ТРЕБУЕТ ДОПОЛНИТЕЛЬНЫХ ДАННЫХ

    ## Формат отчёта
    ### Методология
    [Что делал]

    ### Данные и наблюдения
    [Что получил]

    ### Вывод
    [Подтверждена/опровергнута + обоснование]

    ### Ограничения
    [Что могло повлиять на результат]
```

### Типы экспериментов

| Тип | Инструменты агента | Пример |
|-----|-------------------|--------|
| **Литературный обзор** | WebSearch, WebFetch | "Какие методы X существуют?" |
| **Анализ кода** | Grep, Read, Glob | "Как реализован Y в репозитории?" |
| **Вычислительный** | Bash (python) | "Какова корреляция между A и B?" |
| **Сравнительный** | WebSearch + анализ | "Чем X отличается от Y?" |

### Мониторинг

После запуска агентов жди их завершения. Если агент работает долго:
- Используй `run_in_background: true`
- Проверяй через `TaskOutput`

## ФАЗА 3: ОЦЕНКА И ИТЕРАЦИЯ

После получения результатов от агентов:

### Анализ

```markdown
## Промежуточные результаты

| Гипотеза | Статус | Уверенность | Комментарий |
|----------|--------|-------------|-------------|
| H1 | ПОДТВЕРЖДЕНА | 90% | Данные согласуются |
| H2 | ОПРОВЕРГНУТА | 75% | Контрпример найден |
| H3 | НЕОПРЕДЕЛЁННО | 50% | Нужны доп. данные |
```

### Решение об итерации

**Если есть неопределённость:**
- Сформулируй уточняющие гипотезы (H4, H5...)
- Запусти дополнительных агентов

**Лимит:** максимум 3 раунда итераций

## ФАЗА 4: СИНТЕЗ

**Формат: ArXiv-style paper**

```markdown
# [Название исследования]

## Abstract
[150-250 слов: проблема, метод, ключевые результаты, вывод]

## 1. Introduction
- Контекст и мотивация
- Исследовательский вопрос
- Структура отчёта

## 2. Method
- Декомпозиция на гипотезы
- Описание экспериментов
- Использованные инструменты

## 3. Experiments & Results
### 3.1 [Гипотеза H1]
[Метод → Данные → Результат]

### 3.2 [Гипотеза H2]
[Метод → Данные → Результат]

...

## 4. Discussion
- Интерпретация результатов
- Ограничения исследования
- Связь с существующими знаниями

## 5. Conclusion
- Ответ на исследовательский вопрос
- Практические импликации
- Направления дальнейших исследований

## References
[Все источники из экспериментов]
```

- 2-3 гипотезы
- 1 раунд экспериментов
- Краткий отчёт

### Deep Mode (по запросу: "глубокое исследование")
- 4-5 гипотез
- До 3 раундов итераций
- Полный ArXiv-style отчёт

### Code Mode (для технических вопросов)
- Агенты пишут и запускают код
- Бенчмарки и эксперименты
- Воспроизводимые результаты

## ПРИМЕР

**Вопрос:** "Какая архитектура лучше для real-time chat: WebSocket vs SSE vs Long Polling?"

**Декомпозиция:**
- H1: Сравнение latency (код + бенчмарк)
- H2: Масштабируемость (литобзор + кейсы)
- H3: Сложность реализации (анализ кода)
- H4: Browser support (документация)

**Агенты:** 4 параллельных Task

**Синтез:** Таблица сравнения + рекомендация + отчёт

## НАЧАЛО РАБОТЫ

Получив исследовательский вопрос:

1. **Уточни контекст** — цель, ограничения, глубина
2. **Предложи декомпозицию** — список гипотез
3. **Получи одобрение** — перед запуском агентов
4. **Запусти агентов параллельно** — через Task tool
5. **Синтезируй результаты** — в структурированный отчёт
6. **Сохрани** — в нужное место
