# Анализ созвонов по LARP

Анализ транскриптов мастерских созвонов с системой требований и решений.

## Источник промпта

Полный промпт: `~/09-REFERENCE/my-promts/Игры/Промпт для анализа созвонов по LARP (финальная версия).md`

## Структура анализа

### 1. КРАТКОЕ САММАРИ (100-150 слов)
Основные темы, ключевые решения, прогресс

### 2. СИСТЕМА ТРЕБОВАНИЙ И РЕШЕНИЙ

#### A. ГЛАВНОЕ ТРЕБОВАНИЕ
- **REQ-MASTER**: Целевой опыт игры
- Формулировка: "Создать [описание опыта]. Чем сильнее [метрика], тем лучше."

#### B. ИЕРАРХИЯ ТРЕБОВАНИЙ
- **Уровень 1**: Основные столпы целевого опыта
- **Уровень 2**: Системные механики
- **Уровень 3**: Технические детали

#### C. ПРИНЯТЫЕ РЕШЕНИЯ
- Решения SOL-001, SOL-002 с привязкой к требованиям
- Статусы: ✅ РЕШЕНО / ⚠️ ЧАСТИЧНО / ❌ НЕ РЕШЕНО

### 3. КОНЦЕПТ-ГРАФ (Mermaid)
Визуализация системы требований с цветовым кодированием

### 4. ЛОГИЧЕСКОЕ ДЕРЕВО РЕШЕНИЙ
ASCII-формат иерархии от целевого опыта

### 5. TODO СПИСОК
- Критические требования
- Срочные задачи
- Направления улучшения
- Исследования и эксперименты

### 6. АГЕНДА СЛЕДУЮЩЕГО СОЗВОНА
Приоритеты и временные рамки

## Принципы

- **Честность** — только то, что есть в транскрипте
- **Системность** — все подчинено целевому опыту
- **Направленность** — "Чем... тем лучше" показывает путь улучшения

## Терминология

- immersion, bleed, transparent play
- workshop, debrief, steering
- meta-techniques, runtime, off-game

## Использование

```
/l-g-call
```

Вставь транскрипт созвона после вызова.
